# EX Holon Phantoms (HP) - EngineBench Expansion Spec
# ====================================================

## Overview

Add EX Holon Phantoms as a NEW expansion to engine-bench.
Unlike DF (where engine already supports all mechanics), HP requires
the agent to EXTEND the core engine for new mechanics.

Set Info:
- Name: EX Holon Phantoms
- Release: May 2006 (before Crystal Guardians)
- Cards: 111 total
- Unique mechanic: "Holon's Pokemon" (can attach as Energy)


## Goals

1. Test agent's ability to EXTEND tcg_core (not just use existing hooks)
2. Mix of easy cards (existing hooks) and hard cards (core changes)
3. Gold implementation lives in engine-bench, overzealous stays unchanged
4. Eval via behavior-based integration tests (same as DF)

## Core Engine Changes Required

### Holon's Pokemon Mechanic
Holon's Castform can be attached as a Special Energy card with special conditions:
1. Target Pokemon must already have at least one Energy attached
2. When attached, player must return one Energy from target to hand
3. Provides ALL types of Energy, but only 2 Energy at a time

### New Hooks (tcg_core/src/runtime_hooks.rs)

```rust
/// Check if a Pokemon card can be attached as Energy (Holon's Pokemon)
/// Returns true if the card is a Holon's Pokemon AND target has Energy attached
pub type CanAttachAsPokemonEnergyFn =
    fn(&GameState, card_id: CardInstanceId, target_id: CardInstanceId) -> bool;

/// Get energy types when Pokemon attached as Energy
/// For Holon's Castform: returns all 8 types
pub type PokemonEnergyProvidesFn =
    fn(&GameState, card_id: CardInstanceId, target_id: CardInstanceId) -> Vec<Type>;

/// For Holon's Castform: returns 2 (provides 2 Energy at a time)
pub type PokemonEnergyCountFn =
    fn(&GameState, card_id: CardInstanceId) -> u32;

/// For Holon's Castform: prompts to return an Energy to hand
pub type OnPokemonEnergyAttachFn =
    fn(&mut GameState, card_id: CardInstanceId, target_id: CardInstanceId);

// Add to RuntimeHooks struct:
pub struct RuntimeHooks {
    // ... existing 21 hooks ...

    pub can_attach_as_pokemon_energy: CanAttachAsPokemonEnergyFn,
```

### Game Logic Changes (tcg_core/src/game.rs)

```rust
// Modify can_attach_energy to check Pokemon-as-Energy
pub fn can_attach_energy(&self, card_id: CardInstanceId, target_id: CardInstanceId) -> bool {
    let card = self.get_card(card_id);

    if card.is_energy() {
        return self.validate_energy_attachment(card_id, target_id);
    }

    // NEW: Check if Pokemon can attach as energy (Holon's Pokemon)
    if card.is_pokemon() {
        return (self.hooks().can_attach_as_pokemon_energy)(self, card_id, target_id);
    }

    false
}

// Modify energy_provides to handle Pokemon-as-Energy
pub fn energy_provides(&self, card_id: CardInstanceId, target_id: CardInstanceId) -> Vec<Type> {
    let card = self.get_card(card_id);

    if card.is_energy() {
        return self.standard_energy_provides(card_id, target_id);
    }

    // NEW: Pokemon attached as energy
    if self.is_attached_as_pokemon_energy(card_id) {
        return (self.hooks().pokemon_energy_provides)(self, card_id, target_id);
    }

    vec![]
}

// NEW: Get energy count for attached Pokemon
pub fn pokemon_energy_count(&self, card_id: CardInstanceId) -> u32 {
    (self.hooks().pokemon_energy_count)(self, card_id)
}

// NEW: Handle side effects when attaching Holon's Pokemon as Energy
pub fn on_attach_pokemon_energy(&mut self, card_id: CardInstanceId, target_id: CardInstanceId) {
    (self.hooks().on_pokemon_energy_attach)(self, card_id, target_id);
}
```

### Action Validation (tcg_core/src/action.rs)

```rust
// Modify AttachEnergy action to allow Pokemon cards
Action::AttachEnergy { card_id, target_id } => {
    let card = game.get_card(card_id);

    // Standard energy
    if card.is_energy() {
        return validate_energy_attachment(game, card_id, target_id);
    }

    // NEW: Holon's Pokemon as energy
    if card.is_pokemon() && (game.hooks().can_attach_as_pokemon_energy)(game, card_id, target_id) {
        // Validate target has Energy attached (required for Holon's Pokemon)
        if game.get_attached_energy(target_id).is_empty() {
            return Err(InvalidAction::TargetHasNoEnergy);
        }
        return validate_pokemon_energy_attachment(game, card_id, target_id);
    }

    Err(InvalidAction::NotEnergy)
}
```

## Eval Test Examples

### Tier 1: Core Change Required

```rust
// gold/tests/hp_044_holons_castform_eval.rs

#[cfg(test)]
mod hp_044_holons_castform_eval {
    use super::*;
    use tcg_core::test_utils::*;

    fn setup_with_castform() -> (GameState, CardInstanceId, CardInstanceId) {
        let mut game = GameState::new_test();
        let castform = game.add_to_hand(PlayerId::P1, card_def("HP", 44));
        let pokemon = game.set_active(PlayerId::P1, basic_pokemon());
        (game, castform, pokemon)
    }

    #[test]
    fn test_attach_as_energy_provides_colorless() {
        let (mut game, castform, pokemon) = setup_with_castform();

        // Should be able to attach Castform as energy
        let result = game.apply_action(PlayerId::P1, Action::AttachEnergy {
            card_id: castform,
            target_id: pokemon,
        });
        assert!(result.is_ok());

        // Should provide 2 colorless
        let energy = game.get_attached_energy(pokemon);
        assert_eq!(energy.len(), 1);
        assert_eq!(game.energy_provides(castform, pokemon), vec![Type::Colorless, Type::Colorless]);
    }

    #[test]
    fn test_attach_as_energy_with_holon_provides_all() {
        let (mut game, castform, pokemon) = setup_with_castform();

        // First attach a Holon's Pokemon
        let holon1 = game.add_to_hand(PlayerId::P1, card_def("HP", 44));
        game.apply_action(PlayerId::P1, Action::AttachEnergy {
            card_id: holon1,
            target_id: pokemon,
        }).unwrap();

        // Now attach second Holon's Castform
        game.apply_action(PlayerId::P1, Action::AttachEnergy {
            card_id: castform,
            target_id: pokemon,
        }).unwrap();

        // Second one should provide ALL types
        let provides = game.energy_provides(castform, pokemon);
        assert!(provides.contains(&Type::Fire));
        assert!(provides.contains(&Type::Water));
        assert!(provides.contains(&Type::Grass));
        assert!(provides.contains(&Type::Lightning));
        assert!(provides.contains(&Type::Psychic));
        assert!(provides.contains(&Type::Fighting));
    }

    #[test]
    fn test_can_play_as_basic_pokemon() {
        let (mut game, castform, _) = setup_with_castform();

        // Should be able to play as Basic Pokemon
        let result = game.apply_action(PlayerId::P1, Action::PlayBasic {
            card_id: castform,
        });
        assert!(result.is_ok());

        // Should be on bench
        assert!(game.players[0].bench.iter().any(|s| s.card.id == castform));
    }

    #[test]
    fn test_delta_draw_counts_delta_pokemon() {
        let mut game = setup_game_with_castform_active();

        // Add 3 delta Pokemon to play
        game.add_delta_pokemon_to_bench(PlayerId::P1, 3);

        let hand_before = game.players[0].hand.len();

        // Use Delta Draw
        game.apply_action(PlayerId::P1, Action::Attack {
            attack_name: "Delta Draw".to_string(),
        }).unwrap();

        // Should draw 4 cards (3 bench + 1 active Castform if delta)
        let hand_after = game.players[0].hand.len();
        assert_eq!(hand_after - hand_before, 3); // or 4 if Castform is delta
    }
}
```

```rust
mod hp_007_flygon_eval {

        let mut game = setup_with_flygon_active();

            source_id: flygon_id,
            power_name: "Delta Supply".to_string(),
        }).unwrap();

        // Should prompt for energy selection
        assert!(matches!(
            game.pending_prompt,
            Some(Prompt::ChooseCard { .. })
        ));

        // Select the energy
        game.apply_action(PlayerId::P1, Action::SelectCard {
            card_id: energy,
        }).unwrap();

        // Select target (must be delta Pokemon)
        game.apply_action(PlayerId::P1, Action::SelectPokemon {
            pokemon_id: delta_pokemon,
        }).unwrap();

        // Energy should be attached
        assert!(game.get_attached_energy(delta_pokemon).contains(&energy));
    }

    #[test]
    fn test_delta_supply_blocked_by_special_condition() {
        let mut game = setup_with_flygon_active();
        game.apply_special_condition(flygon_id, SpecialCondition::Asleep);

        // Delta Supply should fail
        let result = game.apply_action(PlayerId::P1, Action::UsePower {
            source_id: flygon_id,
            power_name: "Delta Supply".to_string(),
        });
        assert!(result.is_err());
    }

    #[test]
    fn test_delta_supply_only_targets_delta_pokemon() {
        let mut game = setup_with_flygon_and_mixed_bench();
        let energy = game.add_to_hand(PlayerId::P1, basic_energy(Type::Fire));

        game.apply_action(PlayerId::P1, Action::UsePower {
            source_id: flygon_id,
            power_name: "Delta Supply".to_string(),
        }).unwrap();

        // Select energy
        game.apply_action(PlayerId::P1, Action::SelectCard {
            card_id: energy,
        }).unwrap();

        // Prompt should only show delta Pokemon as valid targets
        if let Some(Prompt::ChoosePokemon { valid_targets, .. }) = &game.pending_prompt {
            for target in valid_targets {
                assert!(game.pokemon_is_delta(*target));
            }
        }
    }
```

## Eval Flow

```
1. Setup sandbox with vanilla overzealous (CG/DF only)

2. Give agent:
   - Stub file (hp_044_holons_castform.rs)
   - Card spec JSON
   - EXTENDING_ENGINE.md documentation
   - CG/DF as reference

3. Agent works:
   - Reads stub, understands requirements
   - Tries existing hooks, realizes they don't support Pokemon-as-Energy
   - Reads EXTENDING_ENGINE.md
   - Adds hooks to tcg_core/src/runtime_hooks.rs
   - Modifies tcg_core/src/game.rs for energy attachment
   - Creates tcg_expansions/src/hp/ directory
   - Implements card logic
   - Runs cargo check, cargo test

4. Eval:
   - Copy agent's changes into fresh sandbox
   - Inject eval tests into card file
   - cargo test --package tcg_expansions hp_044
   - Score = passed / total

5. Scoring:
   - Compile: 30%
   - Tests: 70%
   - Same as DF
```
